# Agent Market Lab — マルチAIエージェント市場分析ツール

複数のAIエージェントが**同時並列**で市場分析を行い、その報告をもとに資料作成エージェントが**完成資料**を自動生成するWebツールです。

## 主な機能

- **マルチエージェント並列分析** — 分析テーマに合わせて最大28体のエージェントチームをAIが自動編成し、同時に分析を実行
- **参考資料・ソースを基にした分析** — WebサイトのURL、テキスト / Markdown / CSV / JSON / HTML / Excel / PDF などのファイルを**複数**追加でき、エージェントは資料を根拠に出典付きで分析。**資料なし（指示文のみ）でも開始可能**
- **SNS風プロフィール** — 各エージェントに名前・役割・アイコンを自動付与。グリッド画面で全員の実行状況（待機中／分析中／報告完了）をリアルタイム表示
- **資料作成エージェント** — 全報告を統合し、エグゼクティブサマリー・戦略提言までまとまった市場分析資料を生成
- **AIプロバイダを自由に選択** — 設定画面で切替可能
  - Claude (Anthropic)
  - ChatGPT (OpenAI)
  - Gemini (Google)
  - Grok (xAI)
  - デモモード（APIキー不要・動作確認用）
- **分析レベル3段階**
  | レベル | エージェント数 | 内容 |
  |---|---|---|
  | Lv.1 クイック | 約8体 | 要点のみ素早く |
  | Lv.2 スタンダード | 約16体 | バランスの良い多角分析 |
  | Lv.3 ディープ | 約28体 | 極めて詳細な徹底分析 |
- **資料のダウンロード** — 完成資料はその場で閲覧でき、**PDF / HTML / Markdown / テキスト** で保存可能
- **自律Webリサーチ** — 細かな指示なしでも、エージェントが市場統計・競合・ニュース・SNS/口コミの傾向を公開Webから自動収集し、出典付きで分析の根拠にします（オン/オフ可）
- **根拠・出典の必須報告** — 各エージェントは報告末尾に「根拠・出典」を明記（資料・自動調査・学習知識・推定を区別）。虚偽データ・架空出典の生成を明示的に禁止するハルシネーション対策済み
- **エラーエージェントの再実行** — エラーになったエージェントはカード内の「再実行」ボタンで個別に再試行でき、完了後は資料を再生成できます（タイムアウト検知・レート制限の自動リトライも搭載）
- **アカウントと端末間同期** — ID・パスワードで登録すると、設定（APIキー含む）はパスワード由来の鍵で暗号化保存。暗号化された同期ファイルを別端末で読み込み、同じID・パスワードでログインすれば同じ設定で利用できます
- **APIキーのセキュリティ** — PBKDF2(310,000回) + AES-GCM 256bitで暗号化。パスワードは保存されず、ログアウトで復号済みデータをメモリから破棄。キーが平文で保存されるのはゲストモードのみ
- **PWA対応（PC / タブレット / スマホ）** — ブラウザから「ホーム画面に追加」「アプリをインストール」するとアプリとして起動可能。レスポンシブ対応でどの画面サイズでも使えます
- **完了プッシュ通知** — 分析が完了すると通知が届くので、待っている間に他の作業ができます（設定でオン／オフ可能）
- **絵文字不使用のピクトグラムUI** — アイコンはすべてSVGピクトグラムで統一

## 使い方

### 1. 起動

ビルド不要の静的Webアプリです。`index.html` をブラウザで開くだけでも動きますが、ローカルサーバー経由を推奨します。

```bash
# Python がある場合
python3 -m http.server 8000

# Node.js がある場合
npx serve .
```

ブラウザで http://localhost:8000 を開きます。（GitHub Pages でそのまま公開することもできます）

### 2. AIの設定

右上の「設定」から使用するAIを選び、APIキーを入力して保存します。
まずはAPIキー不要の**デモモード**で動作を確認できます。

> APIキーはブラウザの localStorage にのみ保存され、選択したAI事業者のAPI以外に送信されることはありません。

### 3. 分析の実行

1. 分析したい市場・商品・課題を入力
2. 必要に応じて参考資料を追加（URL入力、またはファイルをドラッグ＆ドロップ。複数可・省略可）
3. 分析レベル（Lv.1〜Lv.3）を選択し「エージェントチームを起動する」をクリック
4. エージェントたちの分析が完了すると、資料作成エージェント「ツヅリ」が資料を生成
5. 画面下部で資料を閲覧し、PDF等でダウンロード

各エージェントのカードをクリックすると、個別の分析報告を読むことができます。

## クラウド同期のセットアップ（どの端末からでも同じアカウントで利用）

初期状態のアカウントは端末内保存（＋同期ファイルでの移行）ですが、**Firebase（無料）を設定すると、同じID・パスワードでどの端末のブラウザからでも利用できる**ようになります。設定は10分程度・コードの知識は不要です。

### 手順

1. https://console.firebase.google.com/ をGoogleアカウントで開き、「プロジェクトを作成」（名前は任意。Googleアナリティクスは無効でOK）
2. 左メニュー「構築 → Authentication」→「始める」→ ログイン方法で **「メール / パスワード」を有効化**して保存
3. 左メニュー「構築 → Firestore Database」→「データベースを作成」→ ロケーションは asia-northeast1（東京）→ **本番環境モード**で作成
4. Firestoreの「ルール」タブを開き、以下に置き換えて「公開」：
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /users/{uid} {
         allow read, write: if request.auth != null && request.auth.uid == uid;
       }
     }
   }
   ```
5. プロジェクトの設定（歯車アイコン）→「マイアプリ」→ Webアプリ（`</>`）を追加 → 表示される `firebaseConfig = { ... }` をコピー
6. このリポジトリの `js/firebase-config.js` を開き、`window.FIREBASE_CONFIG = null;` をコピーした内容で置き換えてコミット：
   ```js
   window.FIREBASE_CONFIG = { apiKey: "...", authDomain: "...", projectId: "...", ... };
   ```
7. GitHub Pagesに反映されたら完了。アプリの「ログイン」→ 新規登録したアカウントが全端末で共通になります

### Googleアカウントでのログイン（任意）

1. Firebaseコンソール → Authentication → ログイン方法 → 「Google」を有効化（サポートメールを選択して保存）
2. Authentication → 設定 → **承認済みドメイン** に、アプリの公開ドメイン（例: `studiotsuzuri25-debug.github.io`）を追加
3. アプリのログイン画面に「Googleアカウントでログイン」ボタンが表示されます
4. Googleログインの場合も、設定を暗号化する**データ保護用パスワード**（全端末で共通・8文字以上）をパスワード欄に入力してからボタンを押します（初回入力したものがそのアカウントの暗号化パスワードになります）

### セキュリティについて

- パスワードの検証はFirebase Authenticationが行い、パスワード自体は保存されません
- 設定（APIキー含む）は**端末側でAES-256暗号化してから**クラウドに保存されるため、Firebase（Google）側でも中身は読めません
- Firestoreのルールにより、各ユーザーは自分のデータにしかアクセスできません
- `firebaseConfig` は公開されても問題ない識別子です（秘密鍵ではありません）

## アプリとしてインストール（PWA）

HTTPSで公開された環境（GitHub Pages等）または localhost で開くと、アプリとしてインストールできます。

- **PC (Chrome/Edge)**: アドレスバー右端のインストールアイコンをクリック
- **Android (Chrome)**: メニュー →「アプリをインストール」
- **iPhone/iPad (Safari)**: 共有ボタン →「ホーム画面に追加」

通知は初回の分析開始時（または設定保存時）に許可を求めます。iOSでは「ホーム画面に追加」したアプリでのみ通知が利用できます（iOS 16.4以降）。

## 注意事項

- 参考資料を追加しない場合、分析はAIモデルの学習知識に基づきます。数値は推定として扱ってください。
- CORS制限のあるサイトのURLは、公開リーダーサービス（r.jina.ai / allorigins.win）経由で取得されます。機密性の高いURLの追加は避けてください。
- 画像のみで構成されたPDF（スキャンPDF）はテキストを抽出できません。
- 多数のエージェントを並列実行するため、APIのレート制限に達した場合は設定画面で「同時実行数」を下げてください。
- Lv.3 ディープ分析はAPI使用量（コスト）が大きくなります。
